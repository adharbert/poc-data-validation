# Database Schema

SQL Server database for the Customer Validation Portal.

---

## Table Relationships

```
Organizations
    │
    ├── FieldSections          (OrganizationId FK)
    │       │
    │       └── FieldDefinitions (FieldSectionId FK)
    │               │
    │               ├── FieldOptions     (FieldDefinitionId FK)
    │               │
    │               └── FieldValues ◄──────────────────────────┐
    │                                                          │
    ├── Customers                                              │
    │       │                                                  │
    │       ├── FieldValues (CustomerId FK, FieldDefinitionId FK)
    │       │
    │       └── CustomerAddresses (CustomerId FK)
    │               └── CustomerAddresses_History  (temporal)
    │
    └── ImportBatches
            │
            ├── ImportColumnMappings ──► FieldDefinitions
            │       │
            │       └── ImportValueMappings
            │
            ├── ImportErrors
            │
            └── SavedColumnMappings ──► FieldDefinitions
```

## Core Tables

### Organizations
Tenant/business table. Every other table scopes to this.

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK, `NEWSEQUENTIALID()` |
| Name | NVARCHAR(200) | Display name |
| OrganizationCode | NVARCHAR(50) | System-generated ULID |
| FilingName | NVARCHAR(200) | NULL |
| MarketingName | NVARCHAR(200) | NULL |
| Abbreviation | NVARCHAR(4) | **Max 4 chars, unique (filtered)** |
| Website | NVARCHAR(500) | NULL |
| Phone | NVARCHAR(30) | NULL |
| CompanyEmail | NVARCHAR(320) | NULL |
| IsActive | BIT | Default 1 |
| CreateUtcDt | DATETIME2 | |
| CreatedBy | NVARCHAR(200) | |
| ModifiedUtcDt | DATETIME2 | |
| ModifiedBy | NVARCHAR(200) | NULL |

**Constraints:**
- `CK_Organizations_Abbreviation_Length` — `LEN(Abbreviation) <= 4`
- `UQ_Organizations_Abbreviation` — filtered unique index (WHERE NOT NULL)

**Important:** `Abbreviation` is used as the prefix for generated
`CustomerCode` values. Must be set before importing customers.

### Customers
People whose data is being collected. Scoped to an Organisation.
Uses SQL Server **temporal tables** (system versioning) for full history.

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK, `NEWSEQUENTIALID()` |
| OrganizationId | UNIQUEIDENTIFIER | FK → Organizations |
| FirstName | NVARCHAR(30) | |
| LastName | NVARCHAR(50) | |
| MiddleName | NVARCHAR(30) | NULL |
| CustomerCode | NVARCHAR(26) | **System-generated. Never from CSV.** |
| Email | NVARCHAR(150) | NULL. Used for deduplication on import. |
| IsActive | BIT | Default 1 |
| CreatedDt | DATETIME | |
| ModifiedDt | DATETIME | |
| ValidFrom | DATETIME2 | Temporal — hidden, auto-managed |
| ValidTo | DATETIME2 | Temporal — hidden, auto-managed |

**Indexes:**
- `IX_CustomerCode` — UNIQUE on CustomerCode

**CustomerCode generation:**
```
{Organisation.Abbreviation}-{first 10 chars of new ULID}
Example: ACME-01ARZ3NDEK
```
Generated by the API during import. Never mapped from CSV/Excel columns.

The heart of the schema-free design. One row per configurable

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK |
| OrganizationId | UNIQUEIDENTIFIER | FK → Organizations |
| FieldSectionId | UNIQUEIDENTIFIER | FK → FieldSections, NULL |
| FieldKey | NVARCHAR(100) | Internal key e.g. `highest_degree` |
| FieldLabel | NVARCHAR(200) | Display label e.g. "Highest Degree" |
| FieldType | NVARCHAR(20) | See supported types below |
| PlaceHolderText | NVARCHAR(200) | NULL |
| HelpText | NVARCHAR(500) | NULL |
| IsRequired | BIT | Default 0 |
| IsActive | BIT | Default 1 |
| DsiplayOrder | INT | Note: column has typo in DB, keep as-is |
| MinValue | DECIMAL(18,4) | For number fields |
| MaxValue | DECIMAL(18,4) | For number fields |
| MinLength | INT | For text fields |
| MaxLength | INT | For text fields |
| RegExPattern | NVARCHAR(500) | For text fields |
| CreatedDt | DATETIME | |
| ModifiedDt | DATETIME | |

**Supported FieldType values:**
`text` | `number` | `date` | `datetime` | `checkbox` | `dropdown` | `multiselect`

**Constraints:**
- `UQ_FieldDefinitions_Key` — UNIQUE (OrganizationId, FieldKey)
- `CK_FieldDefinitions_Type` — enforces the type list above

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK |
| FieldDefinitionId | UNIQUEIDENTIFIER | FK → FieldDefinitions |
| OptionKey | NVARCHAR(100) | Stored value e.g. `bach` |

### FieldValues

Only the column matching `FieldType` is populated — rest are NULL.

| Column | Type | Used for |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK |
| CustomerId | UNIQUEIDENTIFIER | FK → Customers |
| FieldDefinitionId | UNIQUEIDENTIFIER | FK → FieldDefinitions |
| ValueText | NVARCHAR(MAX) | text, dropdown (stores OptionKey) |
| ValueNumber | DECIMAL(10,4) | number |
| ValueDate | DATE | date |
| ValueDatetime | DATETIME | datetime |
| ValueBoolean | BIT | checkbox |
| ConfirmedAt | DATETIME | NULL = not yet reviewed by customer |
| ConfirmedBy | NVARCHAR(200) | Identity of confirming person |
| FlaggedAt | DATETIME | Set if customer flags an issue |
| FlagNote | NVARCHAR(1000) | Customer's flag description |
| CreatedDt | DATETIME | |
| ModifiedDt | DATETIME | |

**Constraints:**
- `UQ_FieldValues_Customer_Field` — UNIQUE (CustomerId, FieldDefinitionId)

**Trigger:** `trg_field_values_audit` — fires AFTER UPDATE, writes
old row to `FieldValue_History` before every change.

### ImportColumnMappings
One row per CSV column per batch. Defines where each column's
data goes.

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK |
| ImportBatchId | UNIQUEIDENTIFIER | FK → ImportBatches |
| CsvHeader | NVARCHAR(200) | Exact header text from file |
| CsvColumnIndex | INT | Zero-based column position |
| MappingType | NVARCHAR(20) | `customer_field`, `field_definition`, `skip` |
| CustomerFieldName | NVARCHAR(100) | `FirstName`, `LastName`, `MiddleName`, `Email` only |
| FieldDefinitionId | UNIQUEIDENTIFIER | FK → FieldDefinitions, NULL |
| IsAutoMatched | BIT | 1 = system matched, 0 = manually mapped |
| IsRequired | BIT | Copied from FieldDefinition |
| SavedForReuse | BIT | 1 = persist to SavedColumnMappings on success |
| DisplayOrder | INT | |

**Note:** `CustomerCode` is NOT a valid `CustomerFieldName`.
It is always system-generated from `Organisation.Abbreviation + ULID`.

---

### ImportValueMappings
Translates raw CSV values to system OptionKey values for
dropdown and multiselect fields.

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK |
| ImportColumnMappingId | UNIQUEIDENTIFIER | FK → ImportColumnMappings |
| CsvValue | NVARCHAR(500) | Raw CSV value, stored lowercase |
| SystemValue | NVARCHAR(500) | OptionKey e.g. `bach`, or `0`/`1` for checkbox |
| CreatedAt | DATETIME2 | |

**Example mappings:**
```
"Bachelor's Degree"  →  bach
"Masters"            →  mast
"Yes" / "Y" / "1"   →  1   (checkbox)
"No"  / "N" / "0"   →  0   (checkbox)
```

## CustomerAddresses

Tracks every address a customer has ever had.
`IsCurrent = 1` is the live address; all prior rows remain in history.
When a customer moves, the old row flips to `IsCurrent = 0` and a new
row is inserted with `IsCurrent = 1`.

Uses SQL Server **temporal tables** — identical pattern to `Customers`.

| Column | Type | Notes |
|---|---|---|
| Id | UNIQUEIDENTIFIER | PK, `NEWSEQUENTIALID()` |
| CustomerId | UNIQUEIDENTIFIER | FK → Customers |
| AddressLine1 | NVARCHAR(200) | Melissa-standardised when validated |
| AddressLine2 | NVARCHAR(100) | NULL |
| City | NVARCHAR(100) | |
| State | CHAR(2) | |
| PostalCode | VARCHAR(10) | ZIP or ZIP+4 |
| Country | CHAR(2) | Default `US` |
| MelissaValidated | BIT | 1 = Melissa confirmed this is a real, deliverable address |
| CustomerConfirmed | BIT | 1 = customer confirmed this is their correct address |
| IsCurrent | BIT | 1 = active address for this customer |
| CreatedUtcDt | DATETIME2 | |
| ModifiedUtcDt | DATETIME2 | |
| ValidFrom | DATETIME2 | Temporal — hidden, auto-managed |
| ValidTo | DATETIME2 | Temporal — hidden, auto-managed |

**History table:** `CustomerAddresses_History` — created automatically by SQL Server temporal versioning.

**Index:** `IX_CustomerAddresses_CustomerId_IsCurrent` on `(CustomerId, IsCurrent)` — fast current-address lookup.

**Migration:** `scripts/Migrations/Migration_004_CustomerAddresses.sql`

**Mailer eligibility:** rows where `MelissaValidated = 1 AND CustomerConfirmed = 1`.

## Trigger

### trg_field_values_audit
Fires `AFTER UPDATE` on `FieldValues`. Writes the old row to
`FieldValue_History` before every update so a full change trail
is maintained.

Captures: all value columns, `ChangedBy` (from session context),
`ChangedAt`, `ChangeReason` (from session context).

The application sets session context before each update:
```sql
EXEC sys.sp_set_session_context @key = N'current_user_id', @value = N'...'
EXEC sys.sp_set_session_context @key = N'change_reason',   @value = N'...'
```
